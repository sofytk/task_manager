## Постановка задачи

Целью проекта является разработка консольного приложения для управления задачами с поддержкой:

* создания задач
* редактирования и обновления статуса задач
* сортировки задач по приоритету и категории
* фильтрации задач по дате
* сохранения задач

Каждая задача должна содержать:

* дату выполнения
* название
* описание
* категорию
* приоритет
* статус выполнения

Система должна обеспечивать удобное взаимодействие через консольный интерфейс и хранить данные в файле (JSON).

---

## Анализ задачи и структура классов

### Анализ задачи

Для реализации системы управления задачами необходимо:

1. **Хранение данных**
   Используется JSON-файл как простая файловая база данных.

2. **Разделение логики**
   Программа разделена на несколько уровней:

   * работа с данными (Data)
   * бизнес-логика (Domain)
   * пользовательский интерфейс (Presentation)

3. **Моделирование сущностей**
   
   Основная сущность — задача (`Task`), которая содержит все необходимые поля.

4. **Использование перечислений**
   
   Для категорий и приоритетов применяются `Enum`, что исключает ошибки ввода.

---

### Структура классов

#### 1. `Task`

Модель задачи.

**Поля:**

* date — дата
* title — название
* description — описание
* priority — приоритет (Enum)
* category — категория (Enum)
* is_done — статус выполнения

**Методы:**

* `to_dict()` — преобразование в словарь
* `from_dict()` — создание объекта из словаря

---

#### 2. `Category` (Enum)

Содержит категории задач:

* Работа
* Дом
* Семья
* Здоровье и др.

---

#### 3. `Priority` (Enum)

Содержит уровни приоритета:

* высокий
* средний
* низкий

---

#### 4. `BaseTaskRepository` (абстрактный класс)

Определяет интерфейс для работы с данными:

* `get_all_tasks()`
* `get_tasks_by_date(date)`
* `add_task(task)`
* `update_data(task)`
* `delete_task(task)`

---

#### 5. `TaskRepository`

Реализация работы с JSON-файлом.

**Функции:** Наследует от `BaseTaskRepository`

---

#### 6. `TaskController`

Слой бизнес-логики, связывает `TaskRepository` c `Client`

**Функции:**
* `add_task(task)` - добавление новой задачи
* `delete_task(task)` - удаление выбранной задачи
* `update_task_status(task)` - обновление статуса задачи
* `get_all_tasks()` - получение всех задач
* `get_today_tasks()` - получение задач на сегодня
* `get_task_by_date(date)` - получение задач на выбранную дату
* `sorted_by_priority()` - сортировка по приоритету
* `sorted_by_category()` - сортировка по категориям

---

#### 7. `Client`

Консольный интерфейс пользователя.

Функции и методы:
* `show_menu()` – основной цикл меню для выбора действий пользователя.
* `_add_task()` – создание и добавление новой задачи с вводом данных пользователем.
* `_show_all_tasks()` – вывод всех задач из репозитория.
* `_show_today_tasks()` – вывод задач на текущую дату.
* `_show_tasks_by_date()` – фильтрация задач по указанной дате.
* `_sort_by_priority()` – сортировка задач по приоритету с группировкой.
* `_sort_by_category()` – сортировка задач по категориям с группировкой.
* `_mark_task_done()` – пометка выбранной задачи как выполненной.
* `_delete_task()` – удаление выбранной задачи.
* `format_task(task, index=None)` – форматирование задачи в строку для вывода на экран.
* `choose_enum(enum_type)` – выбор значения из перечисления (Priority или Category).
* `input_date(prompt="...")` – запрос и валидация даты у пользователя.
* `display_tasks(tasks)` – вывод списка задач на экран.

## Архитектура проекта

### 1. Presentation Layer (UI)

* `client.py`
* Обработка ввода пользователя
* Вывод данных

### 2. Domain Layer (Бизнес-логика)

* `TaskController`, `Task`, `Category`, `Priority`
* Обработка задач, сортировка, фильтрация

### 3. Data Layer (Данные)

* `BaseTaskRepository`, `TaskRepository`
* Работа с JSON-файлом
